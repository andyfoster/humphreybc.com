Browse the source code on Github!

Monday, 22 October, 2012

I don't think I've mentioned anywhere on my website that the code for this website is in fact available on Github for you to browse and clone.

Essentially this site is put together using Node.js with Brunch (although a lot of stuff has been stripped out). Since there's no logic, all I'm using Node for is for the auto-reload module, Stylus (a CSS precompilier), and some code compression.

I'm also using icon fonts on the homepage, thanks to the Icomoon icon set which makes it super easy to pick what icons you want to use and chuck them in the website.

The font is Museo Slab (if you have it installed), or Arvo as a fallback.

The website runs on an Ubuntu 12.04 Xen VPS hosted with Prgrmr and uses apache2 with it configured to point to the /public directory. There's also a basic cronjob set up that will run "git pull" every 30 minutes so I don't have to SSH in to the server to grab the latest changes I make.

You're welcome to clone and use the code to learn how to easily build your own static website using Node.js, but I do ask that you don't re-use it exactly as it stands for your own site.

Node is a wee bit overkill for this site, but Stylus and the live reload stuff does make building HTML and CSS extremely easy, and it's all very simple to being with one command.

The first blog post.

Sunday, 21 October, 2012

So this is my first blog post. I'm attempting to get a simple blog up and running using the existing site bits and pieces I have with Node.js.

Right now, I have to type my blog post out by hand directly into this HTML file which is a wee bit shit. Ideally, I'd like to have the ability to:

  • Write my posts in markdown
  • Store them in some /articles folder or an articles repository
  • Pull them in and convert them to HTML
  • Add them to my blog.html file
  • Have some basic meta information like a date attached to each
  • Eventually have a unique URL for each blog entry for sharing, etc

So that's that. I have looked into using Wheat, which is what howtonode.org uses. However I can't work out how it actually does things - I cloned the repo and poked around and all I could find was a server.js file which has a little bit of stuff that isn't super helpful.

I did manage to make a simple theme thing for the blog which you can see on the top right. That's kind of nifty.

That's all for now - maybe one day I'll have a proper blog here with proper content! Who knows.
